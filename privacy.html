Database Design

Design a database to store user registration information and lead data. Include tables for users, leads, and any other relevant information.

User Registration

When a user registers on your system, capture their registration date and store it in the database along with other user details.

Lead Management

Store leads in a separate table in the database. Each lead should include information such as lead ID, lead source, lead details, and the date the lead was generated.

Query Data

When a registered user logs in to the system, query the database to retrieve leads based on their registration date. You can use SQL queries to filter leads based on the registration date of the user.

Filtering Previous Leads

Ensure that leads generated before the user's registration date are not included in the data retrieved for that user. You can achieve this by including a condition in your SQL query to filter out leads generated before the user's registration date.

Display Data

Present the filtered leads to the user through your system's interface. This could be in the form of a dashboard, list view, or any other suitable format.

Update Lead Status

As the user interacts with leads, update the status of each lead in the database to reflect the user's actions (e.g., contacted, followed up, converted, etc.).
